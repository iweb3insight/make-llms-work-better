# 与 AI 日常沟通常用关键词

> 日常对话向，侧重"怎么说"而非"是什么"

---

## 1. 输出长度与详细程度

| 关键词 | 使用示例 |
|--------|---------|
| **一句话总结** | "用一句话总结上面的内容" |
| **展开 / 详细说** | "第二点展开说说" |
| **精简 / 压缩** | "太长了，压缩到200字以内" |
| **要点 / 列出要点** | "只列要点，不要解释" |
| **完整版 / 简短版** | "给我完整版" / "给我简短版就行" |
| **不要废话** | "直接给结论，不要铺垫" |

---

## 2. 输出风格与语气

| **通俗解释** | "用初中生能听懂的方式解释" |
| **专业版本** | "改成面向工程师的版本" |
| **换个说法** | "换一种表达方式" |
| **保持原有风格** | "保持我的写作风格，不要改变语气" |
| **幽默 / 严肃** | "加点幽默感" / "改成更严肃的语气" |

## 快速组合套路

```
# 最小有效 Prompt 结构
角色（可选）+ 任务 + 约束 + 格式

# 例子
"你是一个产品经理，帮我写一段电梯演讲，
 面向非技术投资人，不超过100字，用中文"
```

```
# 迭代修改结构
"[保留什么] + [修改什么] + [改成什么样]"

# 例子
"保持整体结构，把语气改得更有说服力，
 结尾加一句行动号召"
```

```
# 质量把关结构
先输出 → "检查一遍，列出你不确定的地方" → 再修正

# 例子
"写完后，主动告诉我哪些信息你不确定"
```

## 高频易忽略的词

这些词单独使用也很有效：

- **"只"** — "只回答问题，不要解释"
- **"不要"** — "不要用列表，用段落"
- **"继续"** — 输出被截断时直接说
- **"等等"** — 打断模型，重新给指令
- **"具体点"** — 回答太抽象时
- **"再想想"** — 质疑答案、要求重新推理
- **"帮我想想还有什么"** — 头脑风暴补充
